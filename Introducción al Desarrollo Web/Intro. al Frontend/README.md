# Introduccion al Frontend

# 🎨 ¿Qué es el Frontend?

El **Frontend (o front end)**, también conocido como **desarrollo del lado del cliente 👩‍💻**,  
es la parte de un sitio web o aplicación web con la que los usuarios **interactúan directamente**.  

👉 Es la **cara visible del sitio** y se encarga tanto de la **interfaz de usuario (UI)** como de la **experiencia del usuario (UX)**.  

---

## 🛠️ Tareas principales del Frontend

- 🏗️ **Diseñar la estructura y el layout** de la página web.  
- ⚙️ **Implementar la lógica de la interfaz de usuario** (cómo reaccionan los elementos en pantalla).  
- 🎛️ **Agregar elementos interactivos** como botones, menús, formularios, animaciones.  
- 📱 **Optimizar el rendimiento** del sitio web para que funcione en diferentes dispositivos y navegadores.  

---

## 🎨 Tecnologías más usadas

- 📝 **HTML** → define la estructura.  
- 🎨 **CSS** → estilos, diseño y animaciones.  
- ⚡ **JavaScript** → interactividad y dinamismo.  

---

## 🌟 Enfocado en UI y UX

- **UI (User Interface)** → lo que el usuario ve (colores, tipografías, botones).  
- **UX (User Experience)** → cómo el usuario se siente al usar el sitio (fluidez, usabilidad, rapidez).  

---

## 📌 Ejemplo sencillo
👉 Cuando entras en una página de e-commerce:  
- El **Frontend** muestra los productos con imágenes, precios y botones 🛒.  
- Podés interactuar con filtros, menús y buscadores 🔍.  
- El objetivo es que la experiencia sea **atractiva y rápida** 🚀.  

---

✨ En resumen:  
El **Frontend es la “cara visible”** de la web, lo que conecta directamente con los usuarios 👀 y define su experiencia 🧑‍💻.

# 🎨 Lenguajes del Front End

Para el desarrollo del **Front End** de una aplicación se suele utilizar un **stack básico de desarrollo**, el cual cuenta con **3 tecnologías principales**:

---

## 🧱 HTML (HyperText Markup Language)
![HTML Logo](https://upload.wikimedia.org/wikipedia/commons/6/61/HTML5_logo_and_wordmark.svg)

- 📌 Es el **lenguaje de marcado** que estructura el contenido de la web.
- Define los **elementos básicos**: textos, títulos, párrafos, imágenes, formularios, enlaces, etc.
- Piensa en HTML como los **ladrillos** que forman la casa de un sitio web.

---

## 🎨 CSS (Cascading Style Sheets)
![CSS Logo](https://upload.wikimedia.org/wikipedia/commons/d/d5/CSS3_logo_and_wordmark.svg)

- 🎭 Es el lenguaje de **estilos** que da diseño y presentación al HTML.
- Controla colores, fuentes, márgenes, posiciones, animaciones y **responsive design**.
- Si HTML son los ladrillos, **CSS es la pintura y decoración** de la casa.

---

## ⚡ JavaScript
![JavaScript Logo](https://upload.wikimedia.org/wikipedia/commons/6/6a/JavaScript-logo.png)

- 🤖 Es un **lenguaje de programación** que da interactividad a la página.
- Permite manejar eventos, validar formularios, actualizar el contenido sin recargar la página (**AJAX**) y conectar con APIs.
- Si HTML son los ladrillos y CSS la pintura, **JavaScript es la electricidad** que enciende todo.

---

✨ En resumen:  
- **HTML** 🧱 → Estructura  
- **CSS** 🎨 → Estilo  
- **JavaScript** ⚡ → Interactividad  

Estos tres forman la **base del desarrollo Front End** 🚀

# 🧱 HTML (HyperText Markup Language)

📌 **HTML** es un **lenguaje de marcado** que define la **estructura del contenido** de una página web.  
Está compuesto por **etiquetas** que sirven para organizar y dividir diferentes partes del contenido.  

Ejemplo de lo que define HTML:
- Títulos (`<h1>`, `<h2>`, …) 📝  
- Párrafos (`<p>`) 📖  
- Imágenes (`<img>`) 🖼️  
- Enlaces (`<a>`) 🔗  
- Formularios (`<form>`) ✍️  

---

⚠️ **Importante**:  
➡️ **HTML *no es* un lenguaje de programación.**  
Es un lenguaje de **marcado** que sirve para **indicar la estructura** de los elementos, pero **no da instrucciones** a la máquina.

---

✨ En resumen:  
- HTML es como los **planos arquitectónicos** de una casa 🏠.  
- Define **qué hay y dónde va**, pero no cómo se ve (eso lo hace CSS) ni cómo funciona (eso lo hace JavaScript).

# 🎨 CSS

**CSS** (*Cascading Style Sheets*) o **Hojas de Estilo en Cascada** es un **lenguaje de hojas de estilo**.  
✨ Te permite aplicar estilos de manera selectiva a elementos dentro de documentos **HTML**.

---

⚠️ **Importante:**  
**CSS no es un lenguaje de programación. Tampoco de marcado.**  
Es un **lenguaje de hojas de estilo**, y su función es dar diseño a elementos que ya fueron previamente indicados y ubicados en una estructura (como HTML).  

📌 Ejemplo:  
- Colores 🎨  
- Fuentes ✍️  
- Espaciado 📏  
- Bordes 🖼️  

Otros ejemplos de lenguajes de estilo son **XSL** y **Sass**.  

---

💡 **Dato curioso:**  
CSS funciona con una cascada de reglas 🪣, lo que significa que si dos reglas entran en conflicto, la última definida (o la más específica) es la que se aplica ✅.  

🌐 Gracias a CSS hoy en día tenemos páginas web modernas, interactivas y visualmente atractivas.

Existen herramientas que ayudan mucho en el desarrollo como Tailwind.

# ⚡ JavaScript

**JavaScript** es un **lenguaje de programación** 🖥️ que se utiliza para añadir **interactividad** a las páginas web 🌐.  

Permite crear:  
- 🎞️ Animaciones  
- 🎮 Juegos  
- 📝 Formularios dinámicos  
- ⚙️ Funcionalidades avanzadas  

Es uno de los lenguajes de programación más **populares del mundo** 🌍 y se utiliza en millones de sitios web.

---

💡 **Dato curioso:**  
JavaScript fue creado en **1995** en tan solo **10 días** por *Brendan Eich* 😱.  
Hoy en día, junto con **HTML** y **CSS**, forma la **triple base del desarrollo web moderno**.

📌 **Ejemplo básico:**

```html
<button onclick="alert('¡Hola desde JavaScript! 🎉')">
  Haz clic aquí
</button>
```

Es el lenguaje más famoso porque es aquel que interpretan los navegadores por defecto.

# 🖼️ Frameworks de Frontend

Los **frameworks de frontend** son herramientas que permiten construir interfaces de usuario de forma más rápida, organizada y escalable. 💡  
Aunque existen muchos, estos son algunos de los más conocidos:

---

## ⚛️ React
- 📌 Creado por **Facebook (Meta)** en 2013.  
- Usa el concepto de **componentes reutilizables**.  
- Muy popular en el desarrollo de **Single Page Applications (SPA)**.  
- 🌍 Tiene una enorme comunidad y ecosistema (Next.js, React Native).

---

## 🅰️ Angular
- 📌 Desarrollado por **Google**.  
- Framework completo (no solo una librería).  
- Utiliza **TypeScript** como lenguaje principal.  
- Ideal para proyectos grandes y corporativos. 🏢  

---

## 💚 Vue.js
- 📌 Creado por **Evan You** en 2014.  
- Combina lo mejor de Angular y React con una curva de aprendizaje sencilla.  
- Popular en Asia 🌏 y en startups por su **simplicidad** y **flexibilidad**.

---

## 🔥 Ember.js
- 📌 Muy utilizado en sus inicios (2011) para aplicaciones web complejas.  
- Ofrece una estructura muy **opinionada** (decide cómo debes organizar tu código).  
- Menos popular hoy en día, pero aún usado en proyectos legacy.

---

## 🟠 Svelte
- 📌 Creado por **Rich Harris** en 2016.  
- A diferencia de otros frameworks, **no necesita un runtime pesado**: el código se transforma en **JavaScript puro** al compilar.  
- Muy rápido ⚡ y cada vez más popular.

---

⚠️ En esta materia **no se profundizará en ninguno de estos frameworks**, pero conocerlos te da una visión de las herramientas modernas en el desarrollo web.

---

💡 **Dato extra:**  
Hoy en día, **React, Angular y Vue** forman el "top 3" más utilizado en frontend.  
Svelte está creciendo rápidamente 🚀, y algunas empresas ya lo adoptan por su eficiencia.

# HTML - Ejemplo de una página

```html
<!DOCTYPE html>
<html>
<head>
    <title>Mi página web</title>

    <!-- Inserto estilos de CSS -->
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <h1 id="titulo">Título principal</h1>

    <!-- Cuerpo de nuestra página Web -->

    <!-- Inserto funcionalidad de Javascript -->
    <script src="script.js"></script>
</body>
</html>
```
